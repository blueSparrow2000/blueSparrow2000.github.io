[A fast reconstruction of the parity-check matrices of LDPC codes in a noisy environment][paperlink]

[paperlink]:[https://arxiv.org/abs/2006.07733](https://www.sciencedirect.com/science/article/abs/pii/S0140366421002152)

LDPC 코드를 공부해볼 기회가 생겨 조사를 하였다. 

목표는 LDPC 코드의 parity check matrix (PCM) 인 H matrix를 찾아내는 것이다. 
입력은 충분히 많은 LDPC 코드 워드 (code word)이고, 해당 코드워드의 길이(n)와 데이터 비트가 차지하는 비트수(k)를 알고있다고 가정한다.
모든 코드는 binary vector이며 패리티 검사 행렬 H의 원소도 전부 이진수이다.

(n,k) LDPC 코드 y는 총 n개의 비트로 이루어져 있으며, 첫 k개가 정보비트이며 에러를 확인하기 위해 redundant information을 가진 parity bit이 나머지 n-k개이다. 
코드는  Hy=0 를 만족하도록 parity bit이 부여되며, Hy가 0이 아니면 에러가 섞였다고 판단할 수 있으며 어느정도의 에러는 복구할수 있다.

code word만 보고 H matrix를 거꾸로 찾아내기 위해서는 Hy=0이라는 성질을 사용한다. 
y가 H의 null space에 속하도록 n-k개의 parity bit이 정해지므로, code word를 모은 행렬 A에 ECO(Elementary column operation)을 적용해 특정 column에 0 vector가 나오게 하는 ECO 행렬의 해당 column이 null space에 속한 parity check vector이다.

즉, noise가 없는 경우 code word를 모은 행렬 A와 ECO
